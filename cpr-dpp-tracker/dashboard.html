Count Integrity

What this table checks: Each family stores hEN/EAD counts in three places that should agree:
  • Actual — counted by iterating standards.standards[] and filtering by type: "hEN" or type: "EAD". This is the ground truth.
  • Range — the value in dpp-range.hen_count / dpp-range.ead_count. Used by the tracker to display DPP range info.
  • Summary — the value in standards.summary.hen_count / standards.summary.ead_count. A convenience field in the standards block.
A red cell means a value disagrees with the Actual count. This typically happens when standards are added or removed from the standards[] array but the numeric counts in dpp-range or summary are not updated to match. Click a row to see all individual standards.
